Introduction

Introduction

    Le profiling est une analyse du programme et se traduit par des mesures de:

  • l'espace mémoire
  • la complexité du programme
  • la fréquence des appels de fonctions
  • la durée de ces appels


    • Cette analyse a pour but d'optimiser le programme.

L'histoire du profiling

Histoire

Les années 70

  • les interruptions d'horloge
  • les simulateurs de jeu d'instruction

Les années 80

  • gprof
    • liste des fonctions utilisées
    • chrono
    • graphe d'appel

Histoire

Les années 90

  • ATOM
    • conversion d'un programme en son propre profiler
    • instrumentation

Les outils du profiling

Les interruptions

Alarme de processus ou interruption matèrielle

  • signal d'alerte électronique
  • asynchrone

Interruption logicielle

  • provoquée par une condition exceptionnelle ou une instruction spéciale
    • pièges/exceptions
    • interrution voulue par le programmeur

L'instrumentation

Plusieurs fonctions :

  • 1 : surveiller ou mesurer le niveau de performance d'un produit
  • 2 : diagnostiquer les erreurs et d'écrire des informations de trace
  • 3 : faire cela automatiquement lors de la compilation


      • n
    • L'instrumentation peut causer des changements de résultats et cela ralentit l'exécution du programme

Le mode Commandes en détails

L'équivalent des menus

Commandes clés :

  • :q : quitter (sans modification)
  • :q! : quitter sans sauvegarder
  • :w : enregistrer le document
  • :wq : enregistrer et quitter (équivalent → ZZ )

Le mode Commandes en détails

L'équivalent des raccourcis claviers/boutons

Manipulation de texte :

  • r : remplace le caractère courant
  • x : efface (coupe) le caractère courant
  • xp : inverse le caractère courant avec le suivant
  • u : annuler la dernière action
  • [N]dd : coupe N lignes à partir de la courante
  • [N]yy : copie N lignes à partir de la courante
  • P : colle la/les lignes mémorisées

Le mode Commandes en détails

Commandes pour le développement

Manipulation de code source :

  • :se ai : active l'indentation automatique
  • :se noai : désactive l'indentation automatique

  • :se nu : active la numérotation des lignes
  • :se nonu : désactive la numérotation des lignes

  • :vsp : coupe l'écran verticalement (2 fichiers côte à côte)
  • :tabedit : ouvre un nouvel onglet

Le mode Commandes en détails

Le mode visuel

Remplace la souris :

  • v : entre en mode visuel


  • Tout déplacement du curseur (flèches directionnelles) modifie la sélection courante
  • La prochaine action réalisée s'applique sur la totalité de la sélection

Exercice 4

Pour chaque élément dans la liste : {pomme poire banane}, créez un répertoire du même nom :

      for i in pomme poire banane; do mkdir $i; done

Exercice 5

Listez tous les fichiers dans votre répertoire personnel portant l'extension .c (ou une autre si vous ne possédez aucun fichier avec cette extension) :

      find ~ -name '*.c'

Listez tous les fichiers dans votre répertoire personnel dont le nom commence par le caractère 't' (en miniscule ou en majuscule) :

      find ~ -iname 't*'

Exercice 6

Listez tous les fichiers dans votre répertoire personnel d'une taille supérieure à 10Ko

      find ~ -size +10k

Créez 5 fichiers toto : toto0 toto1 toto2 toto3 toto4

Listez tous les fichiers dont le nom commence par toto et supprimez les

      find ~ -name 'toto*' -exec rm {} \;

Exercice Bonus

Réalisez un script shell bash affichant un menu permettant d'appeler les exercices précédents

Indices :

  • Placez les commandes des exercices précédents dans des fonctions (mot clé function)
  • Utilisez la fonction getopts

Exercice Bonus

Éléments de solution

      
      exercice="" 
      while getopts abcd option
      do
	 case "$option" in
	 "a") exercice="$exercice exerice1";;
	 "b") exercice="$exercice exerice2";;
	 ...
	 esac
      done

      for i in $exercice; do 
	 eval "$i"
      done

Le mode Commandes en détails

Passer en mode Commandes

  • Passage en mode commande : touche Esc
  • En cas de doute : appuyer plusieurs fois !

Attention ! En mode commande, une seule touche peut faire beaucoup de dégats...

Le mode Commandes en détails

Gestion des modes

Pour revenir en mode insertion :

  • i : insertion à la position courante
  • a : insertion à la position suivante (append)
  • A : insertion en fin de ligne
  • R : mode remplacement
